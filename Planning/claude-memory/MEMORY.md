## Tech decisions (v1)

- **Stack:** static HTML + Tailwind CDN + small vanilla JS. No build step, no npm/yarn. Pages generated by `scripts/build.py` from shared templates + page data.
- **Reason:** npm registry is blocked in our sandbox; static delivery is fastest, simplest, and perfectly suited to a brochure site. Lighthouse scores should be near-perfect.
- **Hosting:** GitHub Pages from a PRIVATE repo (source hidden) with an obscure-URL preview (hotel owner gets the link only).
- **Imagery:** Unsplash-hosted placeholders during demo; swapped for real hotel photos post-review.
- **Languages:** English only for v1. TR/DE/RU are phase-two.

## Lockdown approach (copy-protection)

1. Private GitHub repo (Mi World only collaborator).
2. Obscure GitHub Pages subdomain / path — not linked anywhere.
3. MIT-restricted LICENSE + visible © footer on every page.
4. Minified production HTML (build flag).
5. *Optional phase-two:* Cloudflare Access password gate.

The rendered site's HTML is always visible to any browser — that is inherent to the web and cannot be closed. The above protects the source tree (templates, data, scripts).
