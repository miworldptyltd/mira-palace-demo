## What's here

```
Planning/                  Planning docs, manager report, memory, backlog
scripts/
  build.py                 Generator entry point — renders every HTML page
  common.py                Shared template, nav, footer, helpers, media discovery
  dev_server.py            Local preview server with HTTP Range support
  pages/                   One module per site section
site/
  *.html                   28 generated pages (regenerated on every build)
  assets/css, js, img      Stylesheet, scripts, favicon
  assets/audio, video      Hero videos and ambient music tracks
  assets/js/media-manifest.js   Auto-generated; lists every media file found
.github/workflows/         CI: build and deploy to GitHub Pages on push
```

PowerShell helpers (all self-contained — pick one and run it):

| Script              | What it does                                                  |
| ------------------- | ------------------------------------------------------------- |
| `Run-Dev.ps1`       | Build site, start local preview server on a free port         |
| `Stop-Dev.ps1`      | Stop the running preview server politely                      |
| `Restart-Dev.ps1`   | Stop + rebuild + restart in one step                          |
| `Refresh-Media.ps1` | Re-scan asset folders, rebuild manifest, restart preview      |
| `Audit-Clean.ps1`   | Pre-release housekeeping (run once before tagging a version)  |
| `Init-Repo.ps1`     | First-time GitHub setup: create repo, push, enable Pages      |
| `Publish-Site.ps1`  | Routine: rebuild + commit + push (use after every change)     |

## Stack

- **Static HTML** generated by a small Python build script — no framework, no
  Node, no bundler.
- **Tailwind CSS** via CDN with a custom theme extension.
- **Vanilla JavaScript** (~550 lines) for the customise panel, theme switcher,
  language toggle, video and music players, cross-page music continuation, and
  nav-dropdown hover behaviour.
- **GitHub Pages** for hosting; **GitHub Actions** rebuilds on every push to
  `main`. The dev server (`scripts/dev_server.py`) mirrors GitHub Pages'
  HTTP-Range behaviour locally so audio seek-to-position works the same in dev
  and prod.

## What ships in v0.1.0

- 28 pages across home, about, rooms (4 sub-pages), all-inclusive concept,
  dining (4 sub-pages), pools, spa (2 sub-pages), activities, location,
  gallery, offers, careers, contact, book, three legal pages, and 404.
- 8 hero-video options, 8 ambient-music tracks, picker UI with prev/next
  cluster controls bottom-left (video, home only) and bottom-right (music,
  every page).
- 5 colour themes (Mediterranean default, Sunset Coast, Olive Grove, Midnight
  Pearl, Onyx dark mode) — applies instantly across all pages.
- Bilingual copyright (English / Turkish) switched by the EN / TR flag in the
  nav; visitor's choice persists across pages.
- Music continues playing across page navigations and resumes from the saved
  playback position.
- Customise side-panel slides on hover (mouse) or tap (touch).
- Top-nav dropdowns for Rooms, Dining, Spa with hover-grace behaviour.
- Mobile-responsive: hamburger menu, panel widths capped to viewport, touch
  targets sized for tap.

## Stack the visitor needs

Nothing. Any modern browser on any device — phone, tablet, laptop, desktop —
opens the deployed URL and the site Just Works. No installs, no accounts, no
plugins.

## Quickstart for development

```powershell
# First-time setup (once per machine):
.\Run-Dev.ps1                          # builds and starts a preview server

# After making code or copy edits:
.\Restart-Dev.ps1                      # rebuild and restart preview

# After dropping new media in site/assets/{audio,video}/:
.\Refresh-Media.ps1                    # rescan + rebuild + restart

# Once you're happy locally:
.\Publish-Site.ps1 "what changed"      # rebuild, commit, push to GitHub
```
